Make BAM Files with bwa and samtools

This software uses a lot of memory: indexing the reference need 2 GB of memory; mapping needs more than 2 GB if you have a lot of files to process. So we can do this in two steps:
First, index the reference, download the files, and refresh the page to release the memory for mapping in the next step.
Second, load the indexed files (names with prefix my_index) and all the fastq files, and then run mapping.

I. Index the reference (a fasta file)

Load the reference file and then download the indexed files.



II. Choose demultiplexed fastaq files and the indexed references

You can load several times. If you load something wrong, just refresh the page and restart.

III. Map reads and create bam files

After loading your indexed reference and all the fastq files, now we will use tool subread to create indexed bam files for viewing in software IGV.
